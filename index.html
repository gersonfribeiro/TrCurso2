Aprenda o básico do Git de forma prática e simples!

O que é Git?

Git é um sistema de controle de versões distribuído, amplamente utilizado para gerenciar o código-fonte de projetos de software. Neste curso, vamos abordar os comandos básicos e como usá-los no seu fluxo de trabalho.

Comandos Básicos do Git

1. Inicializando um repositório

O primeiro passo ao usar Git em um projeto é inicializar o repositório. O comando para isso é:

git init

Esse comando cria um repositório Git vazio na pasta onde foi executado. Ele vai permitir que você comece a rastrear mudanças no seu projeto.

2. Adicionando arquivos ao staging

Depois de modificar ou criar arquivos, você precisa adicioná-los ao "staging" (área de preparação). Isso é feito com o comando:

git add

Se você quiser adicionar todos os arquivos modificados de uma vez, use:

git add .

3. Fazendo um commit

O comando git commit serve para registrar as mudanças que você adicionou ao staging:

git commit -m "Mensagem do commit"

Substitua "Mensagem do commit" por uma breve descrição do que foi alterado no código.

4. Verificando o status do repositório

Para ver quais arquivos estão sendo rastreados, se há mudanças não comprometidas, e o estado geral do repositório, você pode usar:

git status

5. Visualizando o histórico de commits

Se você quiser ver o histórico de commits no repositório, use o comando:

git log

Isso vai mostrar todos os commits feitos até o momento, incluindo as mensagens de commit e IDs de commit.

6. Criando e mudando de branch

Para trabalhar em diferentes funcionalidades sem interferir no código principal, você pode criar e mudar de branches:

git branch nome_da_branch

Para mudar para a branch criada:

git checkout nome_da_branch

7. Adicionando um repositório remoto (git remote add)

Se você quiser conectar seu repositório local a um repositório remoto (por exemplo, no GitHub), você usa o comando git remote add.

git remote add origin

Substitua <URL_do_repositório_remoto> pela URL do seu repositório no GitHub, GitLab, Bitbucket ou qualquer outro serviço de hospedagem de repositórios. O nome mais comum para o repositório remoto é origin.

8. Verificando o repositório remoto (git remote -v)

Para verificar se o repositório remoto foi configurado corretamente, você pode usar o comando:

git remote -v

Esse comando vai listar a URL do repositório remoto, tanto para o fetch quanto para o push, e confirmará se o repositório está corretamente configurado.

9. Enviando mudanças para o repositório remoto (git push)

Depois de fazer commits localmente, você pode enviar as mudanças para o repositório remoto, como o GitHub. O comando para isso é:

git push origin nome_da_branch

O comando git push envia suas mudanças para o repositório remoto. Substitua nome_da_branch pelo nome da sua branch (geralmente main ou master).

Se você estiver enviando pela primeira vez, pode ser necessário fazer a associação da sua branch local com o repositório remoto, usando:

git push -u origin nome_da_branch
